Types de donnees

Reprenons les exemples vus avec le REPL. 

1+1
2

1.0+1.0
2.0

Pour l'addition, on peut voir deux comportements differents. Ils correspondent a deux types de donnees differentes.

Dans le premier cas, Kotlin interprete 1 comme un integer. La somme de deux int est elle aussi un entier. Le resultat est alors affiche comme un entier, donc 2.

Dans le 2eme cas, Kotlin interprete 1.0 comme un floating point. Il fait alors la somme et retourne 2.0, qui est lui aussi un float . Jusque la, il n'y a pas de grande difference.

On voit un comportement similaire avec la division. Pour les entier c'est la partie entiere de la division, pour les float c'est la division avec la partie decimale. 

  79/2
39

79.0/2.0
39.5

Les chaines de characteres

Les string sont le type de donnees utilise pour representer du texte. Parce qu'il sagit de texte, on ne va pas pouvoir faire les memes operations qu'avec des entiers. L'operation la plus courante avec les strings est la concatenation. Pour ca on utilise l'operateur +. 

  "La vache qui rit, " + "a moitie dans ton lit"
"La vache qui rit, a moitie dans ton lit"

Ce + est different du + de l'addition parce qu'il s'applique sur une string et autre chose. Heureusement en Kotlin cette autre chose peut etre n'importe quelle valeur. Kotlin va toujours se debrouiller pour representer textuellement un element. 

  "1 + 1 = " + ( 1 + 1 )
1 + 1 = 2
