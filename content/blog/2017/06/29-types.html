Lorsque l'on code, on aime bien savoir ce qu'on a dans les mains. Programmer c'est souvent suivre des recettes de cuisine et reutiliser les recettes des autres. Encore faut-il savoir quels ingredients utiliser et savoir quels ingredients on a a disposition.

Est ce que je peux faire un calcul avec ca, est ce que je peux l'afficher, est ce que ca va exploser si j'ai met le feu ? Autant de questions auquelles le typage de donnees va tenter de repondre.

Les types de donnees

Il existe une tres grande variete de types de donnees differents. Pour etre plus precis, il y en a plusieurs milliers fournies en standard avec Kotlin et des dizaines de milliers d'autres provenants de contributeurs. Cette variete est permise grace a l'intercompatibilite de Kotlin avec Java qui permet de reutiliser tout le code java deja existant. Encore une fois, c'est juste une boite a outils. Ca ne sert a rien de tous les connaitres et nous verrons plus tard comment s'en servir. Revenons donc a nos moutons types.

Reprenons les exemples vus avec le REPL. 

1+1
2

1.0+1.0
2.0

Pour l'addition, on peut voir deux comportements differents. Ils correspondent a deux types de donnees differents.

Dans le premier cas, Kotlin interprete 1 comme un integer. La somme de deux int est elle aussi un entier. Le resultat est alors affiche comme un entier, donc 2.

Dans le 2eme cas, Kotlin interprete 1.0 comme un floating point. Il fait alors la somme et retourne 2.0, qui est lui aussi un float . Jusque la, il n'y a pas de grande difference.

On voit un comportement similaire avec la division. Pour les entier c'est la partie entiere de la division, pour les float c'est la division avec la partie decimale. 

  9/2
9

9.0/2.0
4.5

Les chaines de characteres

Les string sont le type de donnees utilise pour representer du texte. Parce qu'il sagit de texte, on ne va pas pouvoir faire les memes operations qu'avec des entiers. L'operation la plus courante avec les strings est la concatenation. Pour ca on utilise l'operateur +. 

  "La vache qui rit" + ", a moitie dans ton lit"
"La vache qui rit, a moitie dans ton lit"

Ce + est different du + de l'addition parce qu'il s'applique sur une string et autre chose. Heureusement en Kotlin cette autre chose peut etre n'importe quelle valeur. Kotlin va toujours se debrouiller pour representer textuellement un element. 

  "1 + 1 = " + ( 1 + 1 )
1 + 1 = 2

Les booleens

Les booleens sont le type de donnees le plus simple. Un Boolean peut etre soit vrai, soit faux. C'est ce type de donnees qui est utilise en logique, avec les operateurs

  • &&, and
  • ||, or
  • ==, equals

Les variables et les types

Pour savoir ce que l'on a et ce que l'on peut faire avec une variable, ces dernieres sont typees. Cela veut dire que pour chaque variable ou valeur declaree, on lui associe une 2eme information qui est ce que la variable represente. Cela permet deux choses essentielles:

  • Ca permet a Kotlin de verifier avant meme l'execution qu'on donne le type demande
  • Ca permet a IntelliJ d'informer le developpeur a propos du type de donnees qui peut etre utilise
